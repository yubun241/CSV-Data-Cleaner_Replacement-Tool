## CSV Data Cleaner - BURNOUT Replacement Tool
このプロジェクトは、複数の工場・設備から出力されるCSVデータを自動的に巡回し、特定の異常値文字列をデータ分析に適した形式（NULL/NaN）に一括置換して保存するPythonスクリプトです。

## 概要

工場内のネットワーク共有フォルダに保存されるCSVファイルを監視し、以下の処理を自動化します。

## 主な機能

処理フローの再構築:
まず error_bk 内のファイルを本来の保存先（base）へ shutil.move で移動。
移動完了後、base ディレクトリ内の全ファイルを対象に一括で Pandas 置換処理を実行。
設定リストの統合:
通常処理と救済処理のパス情報を一つの辞書（TARGET_CONFIGS）で管理するように集約。
移動（Cut & Paste）の採用:
コピーではなく「移動」を行うことで、処理後に error_bk フォルダ内に古いファイルが残らないように改善。

## フォルダ構成（検証環境）

検証時は以下のディレクトリ構造を想定しています。

```text
.
├── src/
│   └── main.py         # 実行スクリプト
├── data/               # 通常監視対象のCSVを配置/置換後の出力先
└── dataerror/          # エラー救済対象のCSVを配置
```

## 技術仕様

### 動作環境
- Python 3.9+
- Pandas
- Numpy

### 処理ロジック
| 項目 | 内容 |
| :--- | :--- |
| **文字コード** | `UTF-8` (Shift-JIS) |
| **置換内容** | `BURNOUT` → `NaN` (空値) |
| **読み込み設定** | `low_memory=False`（大規模データによる型推論エラーを防止） |
| **ファイル保護** | 読み込みに失敗したファイルは保存（上書き）されず、元の状態で維持されます。 |

## 使い方

1. 必要なライブラリをインストールします。
   ```bash
   pip install pandas numpy
   ```
2. `targets` リスト内のパスを、実際の環境に合わせて書き換えます。
3. スクリプトを実行します。
   ```bash
   python main.py
   ```
