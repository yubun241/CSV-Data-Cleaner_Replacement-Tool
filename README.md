## CSV Data Cleaner - BURNOUT Replacement Tool
このプロジェクトは、複数の工場・設備から出力されるCSVデータを自動的に巡回し、特定の異常値文字列をデータ分析に適した形式（NULL/NaN）に一括置換して保存するPythonスクリプトです。

## 概要

工場内のネットワーク共有フォルダ（KEYENCE/OMRON等）に保存されるCSVファイルを監視し、以下の処理を自動化します。
1. **通常処理**: 監視フォルダ内のCSVから異常値を削除して上書き保存。
2. **エラー救済処理**: エラー用バックアップフォルダ（`error_bk`）からファイルを回収し、クレンジングした上で正常なフォルダへ復旧。

## 主な機能

- **一括置換**: CSV内の `BURNOUT` という文字列を `NaN` (空値) に置換します。
- **堅牢なエラーハンドリング**: 
  - 特定のファイルが読み込みエラー（ファイルロックや破損）を起こしても、処理を中断せず次のファイルを処理します。
  - `BURNOUT` という文字列が含まれていないファイルも、スキップせず正常に処理・保存されます。
- **マルチターゲット対応**: リスト形式で複数のパス（UNCパス対応）を一度に定義可能です。
- **自動フォルダ作成**: 保存先ディレクトリが存在しない場合、自動的に作成します。

## フォルダ構成（検証環境）

検証時は以下のディレクトリ構造を想定しています。

```text
.
├── src/
│   └── main.py          # 実行スクリプト
├── data/                # 置換後の出力先
├── data/               # 通常監視対象のCSVを配置
└── dataerror/           # エラー救済対象のCSVを配置
```

## 技術仕様

### 動作環境
- Python 3.9+
- Pandas
- Numpy

### 処理ロジック
| 項目 | 内容 |
| :--- | :--- |
| **文字コード** | `UTF-8` (Shift-JIS) |
| **置換内容** | `BURNOUT` → `NaN` (空値) |
| **読み込み設定** | `low_memory=False`（大規模データによる型推論エラーを防止） |
| **ファイル保護** | 読み込みに失敗したファイルは保存（上書き）されず、元の状態で維持されます。 |

## 使い方

1. 必要なライブラリをインストールします。
   ```bash
   pip install pandas numpy
   ```
2. `targets` リスト内のパスを、実際の環境に合わせて書き換えます。
3. スクリプトを実行します。
   ```bash
   python main.py
   ```
